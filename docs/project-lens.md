# Trinity's project-lens

> **What this is:** the *descriptive* mirror of the user-taste lens Trinity already
> builds — but extracted from Trinity's **own** corpus: its git fix-history and its
> regression-guard families. Same hierarchy (invariants → tensions → basins →
> vocabulary), same accretion idea (each tension carries a **support count** = how
> many distinct fixes/guards instantiate it).
>
> **What this is NOT:** a runtime gating system. The dogfood council
> (`council_c2d959d1496bf6b2`, winner: claude) explicitly rejected building this as
> new infrastructure — "kill the overbuilt lens, keep the invariant as executable
> guards" — judging an LLM-advisory hierarchy to be moves-substrate-2.0 (the layer
> retired in #184). So this doc is a **map you read**; enforcement lives in the
> concrete CI guards each tension links to. Generated by the `project-lens-extract`
> + `guard-coverage-gap` dogfood workflows, 2026-05-29.

## Invariants (the core layer — non-negotiable)

1. **No LLM calls outside councils.** Ingest, embedding, theme assignment, ranking, clustering = pure embeddings + heuristics + metadata. Only council member + chairman synthesis invoke a model, riding the user's subscriptions.
2. **Prompt content never leaves the machine; inference is local-first.** Only categorical routing labels may ship (anonymous, opt-in-config). No hosted controller, no per-call billing, no API key. HF Hub pinned offline at startup.
3. **Every live claim resolves to code.** Every numeric count, named artifact, command, commit ref, schema field, and brand string in a `class: live` surface must resolve against the code/git/config it describes — canonical value in exactly one guarded place, all surfaces agreeing.
4. **The single write-point refuses a degenerate clobber.** A save must not overwrite a populated store with an empty/degenerate result (empty when ≥5 rows, or below ~25%): stash to a `.degenerate` sidecar, raise, abort *before* downstream stages overwrite siblings. `allow_shrink` is the only escape hatch.
5. **Recorded == dispatched.** Reconcile at load so the actually-used model/effort/id wins and is what gets persisted (lift inline `--model` into `config.model`; content-hash `stable_id` over batch-local sequence ids) *before* any benchmark or routing claim is made.
6. **Validate on the real corpus + realistic backends.** TF-IDF fallback, sanitized PATH, `file://`, real Chrome/provider subprocesses. A semantic filter must degrade to count-only when MLX isn't actually loaded; every dual-method dataclass must round-trip losslessly.

## Tensions (sorted by support — the recurring failure-mode pairs)

### ⓘ 117 · Doc/count/claim drift from shipping reality · `doc-consistency`
- **the easy wrong move:** hardcode a count / schema-field list / command / commit SHA / brand line inline, fix each drift one-shot when an audit happens to catch it.
- **the invariant:** drive every cross-cutting claim from one canonical source pinned by an automated doc-consistency guard added the *same* iter the drift is fixed.
- **enforced by:** the 108 guards in `tests/test_doc_count_consistency.py` (`test_four_surfaces_agree`, `test_claude_md_lists_all_mcp_server_tools`, …). **Still drifting:** retired `rejections.jsonl`/`decisions.jsonl` paths shown as live across docs + SKILL.md + `eval-import --help` (WF-3).

### ⓘ 21 · Per-provider dispatch fidelity (recorded == dispatched) · `provider-dispatch`
- **the easy wrong move:** inject the effort/model flag conveniently — or let a refactor drop/double it, leak `--model` into agy, trust batch-local `r_001` ids, or record `config.model` while args dispatch another model.
- **the invariant:** translate effort/model into each CLI's exact dialect, inject once in correct order before `-p`; reconcile inline `--model` into authoritative `config.model`; content-hash `stable_id()` so recorded == executed.
- **enforced by:** `tests/test_provider_effort_injection.py` (incl. the new `test_no_model_flag_injected_for_antigravity` — agy must be exactly `['agy','-p',<prompt>]`), `_reconcile_model_arg`. **This is the family the v1.7.60 `--model` regression belonged to.**

### ⓘ 8 · Mock/synthetic tests green while the real backend fails · `test-rigor`
- **the easy wrong move:** validate against mocked `embed()` / orthogonal synthetic vectors / degenerate fixtures — tests stay green while the code is inert at production scale.
- **the invariant:** run on the real corpus + realistic backends; check `mlx_actually_loaded()` and degrade to count-only on TF-IDF; vary fixtures along production dimensions; pair every string-presence assert with a real-browser/real-backend smoke.
- **enforced by:** `tests/test_real_corpus_invariants.py`, the 34-surface `browser_smoke.py`. **Source:** #185/#186 (TF-IDF fallback silently gutted the lens; synthetic tests passed because they mocked `embed()`).

### ⓘ 4 · Degenerate-extraction clobber of a populated store · `persistence`
- **the easy wrong move:** let a transient/empty/partial upstream stage persist its near-empty result and report `ok:true`, gutting the corpus/ledger/lens.
- **the invariant:** the single truncation point refuses the cliff-drop, stashes a `.degenerate` sidecar, raises, aborts before siblings get overwritten.
- **enforced by:** `save_preference_acts` clobber guard (#194/#203/#219). **Gap (WF-2):** the guard was applied to *one* store — `save_registry`, `save_basins`, `save_lenses`, `save_routing_patterns`, and the `lens.md` empty-output branch are still unguarded.

### ⓘ 4 · Format/path migration leaves new code reading the empty old shape · `persistence`
- **the easy wrong move:** flip a store's format/header/filename, let the consumer keep reading the legacy shape → silently returns 0 on a populated install.
- **the invariant:** ship the migration in the same change — read modern-first with legacy fallback, idempotent, invoked at every entry point; verify on the live populated install.
- **enforced by:** `_migrate_legacy_preference_stores()` (#219). **Source:** the `0/0` memory-compare + `0 captures` Gemini-sidecar bugs.

### ⓘ 1 · Substring-presence asserts survive partial reverts · `test-rigor`
- **the easy wrong move:** a regression test checks a single substring's presence → stays green after a partial revert because the string survives in orphaned code.
- **the invariant:** structural assertions (declaration site + increment site + constant); run the revert-test-restore loop to confirm the test actually fires.

## Vocabulary (canonical vs retired naming — pin these)

- **Slugs in code/config/paths/JSON** (`claude`, `codex`, `antigravity`); **model names in UI** (Claude Opus 4.8, GPT-5.5, Gemini 3.x).
- **Primary CLI verbs:** `lens` / `council` (#213). Aliases `lens-build` / `council-launch` survive only for launchpad dispatch + extension allowlist.
- **Canonical stores:** `prompts/` (not `memory/`), `todos/` (not `tasks/`), `preference_acts.jsonl` as the SOLE unified ledger (`rejections.jsonl` + `decisions.jsonl` retired).
- **Canonical modules:** `launchpad_*.py` (not `portal_*.py`), `task_type` (not `task_kind`), `health_checks.py` (the `doctor` verb was retired).
- **Never present-tense as live:** handoff, memory_compare, record_outcome, get_eval_summary, the moves substrate, replay-history, the autofill suggestion panel.
- **core memories** = the three thinking files: `lens.md` (paired tensions), `topics.json` (subject basins), `vocabulary.md` (linguistic anchors).
- **member** = a provider as one council voice; **chairman** = the synthesis model; **Conductor** = the session-level model picker.
- Pin facts via `canonical:<name>` placeholders, never raw numbers in prose. CHANGELOG may be stale; `class: live` / status-block claims may not.
