# Code of Conduct

Trinity Local is a small, focused open-source project. Our code of conduct is short:

## The rules

1. **Be technical.** Ground claims in code, data, or council outcomes. "I think X is bad"
   is less useful than "X breaks Y in scenario Z; here's a reproducer."

2. **Be specific.** When you disagree, name what specifically you'd change. PRs > issues
   > comments. The council mechanism Trinity ships is built around the idea that
   *named disagreement is more valuable than agreement.* That applies to humans too.

3. **Respect the architectural commitments.** Privacy posture, no LLM outside councils,
   local-first inference, free-forever v1 — these are non-negotiable. PRs that violate
   them get rejected with a pointer to `claude.md` or `docs/spec-v1.md`. Not personal,
   just policy.

4. **No harassment.** No personal attacks, no off-topic political flame wars in issue
   threads, no doxxing, no targeted harassment of contributors or maintainers.
   Repeated violations get you banned from the repo's interaction surface.

5. **Be patient with maintainers.** This is a free open-source project. Response times
   vary. If something is urgent or security-sensitive, see `SECURITY.md`.

## Reporting

If a contributor's behavior violates this code of conduct, email
**conduct@openclaw.dev** with details. Reports stay confidential; we take action
proportional to the offense.

## Coverage

This applies to all Trinity Local project spaces — GitHub repo, issues, pull requests,
discussions, any official Discord/Slack/etc. that ships in future.

## In one line

*Be technical, be specific, respect the commitments, no harassment, be patient.* If
your contribution fits that frame, you're welcome here.
